Méthodes de notification —

Configure ici le destinataire de chaque méthode de notification et active/désactive globalement la méthode. Seul l'e-mail est opérationnel ; les autres méthodes (MQTT, SMS, Slack) sont des emplacements réservés pour une évolution future.

Méthode État Activer Destinataire Test
Chargement…
Le bouton Tester envoie un message de test via la méthode (vers le destinataire saisi), sans rien enregistrer. Il n'est actif que pour une méthode disponible et activée.

Alarmes à notifier

Pour chaque alarme détectée par le service de supervision, choisis les méthodes par lesquelles une notification doit être envoyée. Une alarme est émise dès que l'évènement correspondant est enregistré (les tâches de relevé détectent le démarrage/arrêt de Mirth, l'arrêt/redémarrage du checker et le démarrage du système).

Alarme Gravité
Chargement…
Le bouton Tester de chaque ligne déclenche l'envoi multiple de l'alarme via les méthodes actuellement cochées (état affiché, même non enregistré). Il n'enregistre rien en base — aucun évènement ni relevé n'est créé (utile pour le test et le développement).
